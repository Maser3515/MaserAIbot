# ChatGPT для работы – Telegram-бот курса

Этот репозиторий содержит исходный код Telegram-бота, который продаёт и выдаёт цифровой курс **"ChatGPT для работы"**. Бот написан на Python&nbsp;3.11 с использованием фреймворка **aiogram** и использует SQLite для хранения данных. Все тексты вынесены в отдельный модуль, а клавиатуры – в отдельный файл.

## Возможности бота

- Приветствие и регистрация пользователя при `/start` с сохранением параметра deep&nbsp;link.  
- Бесплатный лид‑магнит: PDF с уроком «7 ошибок, из‑за которых ChatGPT даёт слабые ответы»【14†L3-L10】.  
- Описание программы курса и продающее сообщение.  
- Оплата курса (режим **manual_link** или задел под **webhook**). В базовом режиме пользователь видит ссылку или инструкции по оплате и отправляет чек администратору.  
- Админ получает заявку и выдаёт доступ командой `/add_access USER_ID`.  
- После оплаты пользователю открываются пять модулей с текстовыми уроками и PDF‑файлами.  
- Пользователь может скачать PDF‑материалы, оставить отзыв и написать в поддержку.  
- Админ‑панель с командами для просмотра статистики, пользователей, рассылок и экспорта CSV.

## Подготовка и запуск

1. **Создайте бота в BotFather.** Получите токен и запишите его.  
2. **Узнайте свой ADMIN_ID.** Это числовой ID аккаунта Telegram, который будет получать заявки и пользоваться админ‑командами. Его можно узнать, написав своему боту и посмотрев `message.from_user.id` в логах.  
3. **Скопируйте `.env.example` в `.env`** и заполните значения:

   - `BOT_TOKEN` – токен бота.
   - `ADMIN_ID` – ваш Telegram user ID.
   - `PAYMENT_MODE` – `manual_link` для ручной проверки (по умолчанию).
   - `PAYMENT_URL` – ссылка на страницу оплаты (если используете внешнюю платёжную форму) или оставьте пустой, чтобы бот отправил инструкцию по оплате, указанную в `content.py`.
   - `PAYMENT_PROVIDER_NAME` – название платёжного сервиса (будет записано в базу).
   - `DB_PATH` – путь до файла базы данных. По умолчанию `bot.db`.

4. **Поместите PDF‑файлы в папку `materials/`.** В репозитории уже есть примеры (бесплатный урок и модули). Если вы обновляете материалы, соблюдайте имена файлов:

   - `ChatGPT_dlya_raboty_besplatny_urok.pdf` – бесплатный урок;
   - `ChatGPT_dlya_raboty_polny_kurs.pdf` – полный курс;
   - `ChatGPT_dlya_raboty_modul_1.pdf` … `modul_5.pdf` – отдельные модули.

5. **Установите зависимости:**

   ```bash
   python3 -m venv venv
   source venv/bin/activate
   pip install -r requirements.txt
   ```

6. **Запустите бота:**

   ```bash
   python bot/main.py
   ```

Бот будет работать в режиме long&nbsp;polling. Для продакшена можно настроить webhook.

## Тестирование функциональности

1. **/start.** Напишите `/start` вашему боту. Он зарегистрирует вас в базе, сохранит параметр deep&nbsp;link (если есть) и покажет главное меню.  
2. **Бесплатный урок.** Нажмите «Получить бесплатный урок». Бот отправит PDF лид‑магнита и текст с пользой. В базе появится событие `free_lesson_sent`.  
3. **Описание курса.** Кнопка «Что внутри курса» покажет описание программы и цену.  
4. **Покупка.** Кнопка «Купить курс за 499 ₽» откроет продающее сообщение. Затем нажмите «Перейти к оплате». При `PAYMENT_MODE=manual_link` бот вышлет ссылку или инструкцию по оплате и кнопку «Я оплатил».  
5. **Ручная проверка оплаты.** После оплаты нажмите «Я оплатил» и отправьте чек/скриншот. Бот перешлёт подтверждение админу и поблагодарит пользователя. Админ выдаёт доступ командой `/add_access USER_ID`.  
6. **Доступ к модулям.** После подтверждения оплаты бот покажет кнопки для Модуля 1. При выборе модуля бот высылает текст урока и PDF соответствующего модуля.  
7. **Материалы.** Кнопка «Мои материалы» позволяет скачать полный курс и отдельные PDF, а также перейти к модулям.  
8. **Отзывы.** После завершения всех модулей появится кнопка «Оставить отзыв». Бот попросит написать текст, затем предложит прикрепить фото. Отзыв отправляется админу.  
9. **Поддержка.** В любой момент можно нажать «Поддержка» и отправить вопрос – он придёт админу.  
10. **Админ‑команды.** Введите `/admin` для списка команд. Можно посмотреть статистику `/stats`, список покупателей `/buyers`, информацию о конкретном пользователе `/user USER_ID`, выдать доступ `/add_access USER_ID`, удалить доступ `/remove_access USER_ID`, отклонить оплату `/deny_payment USER_ID`, сделать рассылку `/broadcast текст` или только покупателям `/broadcast_buyers текст`, посмотреть отзывы `/reviews` и выгрузить CSV `/export_users` или `/export_buyers`.

## Сборка проекта в ZIP

Чтобы собрать проект в архив, выполните из корня репозитория:

```bash
zip -r ChatGPT_dlya_raboty_bot.zip bot/
```

Этот архив будет содержать всю структуру проекта. После распаковки вам нужно будет только заполнить `.env` и положить PDF‑файлы в папку `materials`.

## Что нужно заполнить при запуске

- **BOT_TOKEN** – токен Telegram‑бота;  
- **ADMIN_ID** – ваш Telegram ID;  
- **PAYMENT_URL** – ссылка на оплату (если используете форму);  
- **PAYMENT_PROVIDER_NAME** – название платёжного сервиса;  
- **PDF‑файлы** – положите в `bot/materials/` файлы курса и модулей.

## Подготовка к webhook‑оплате

Код предусматривает режим `PAYMENT_MODE=webhook`. Для внедрения автоматических оплат нужно:

1. Подключить платёжного провайдера (Robokassa, ЮKassa и т.п.) и создать заказ в функции `create_payment`.  
2. Настроить HTTPS‑сервер или облачную функцию для приёма webhook‑запросов.  
3. Реализовать в `services/payments.py` проверку подписи и обновление статуса платежа через `database.update_payment_status`.  
4. После подтверждения оплаты автоматически вызывать `database.update_user_access(..., True)` и уведомлять пользователя.

Эти шаги требуют знаний API выбранной платёжной системы и серверного окружения с HTTPS.

## Источники вдохновения

Материалы курса и бесплатного урока были созданы на основе документов, предоставленных пользователем. Например, один из модулей подробно описывает, как использовать ChatGPT для учёбы и личных задач, и содержит 25 промтов【14†L3-L10】. Вся структура курса и тексты берутся из этих PDF‑материалов.
